你正在执行 /project:create-task。

目标： 将当前对话中「plan / plant 模式」下的输出内容， 整理并写入一个 task/ 下的
task 文件。

严格遵守以下规则：

【输入来源】

- 仅使用当前对话中已经产生的计划 / 分析 / 需求讨论内容
- 不引入新的实现细节
- 不假设尚未讨论的技术方案

【禁止事项】

- 不修改任何 src/** 下的代码
- 不执行 lint / test / build
- 不做实现、不写伪代码

【输出文件】

- 路径：task/<filename>.md
- 如果用户未指定文件名：
  - 使用当天日期 YYYY-MM-DD
  - slug 从“目标”自动生成

【Task 文件必须包含以下结构（按顺序）】

# 目标

- 一句话描述本任务要解决什么问题

# 背景 / 上下文

- 需求来源
- 相关模块、页面、业务背景
- 已知限制或现状

# 需求范围

## 包含

- 明确需要做的事情（列表）

## 不包含

- 明确不在本任务内的内容（防止 scope creep）

# 验收标准（必须可验证）

- [ ] 明确、可测试、可判断对错的条件
- [ ] 避免模糊词（如“优化”“更好”）

# 影响范围

- 预计改动目录 / 模块
- 关键组件（如果涉及，写明组件名）

# 约束与规范

- 必须遵循的 CLAUDE.md / 组件规范
- 不允许的行为（如：不升级依赖、不改接口）

# 备注

- 风险
- 未决问题
- 后续可能拆分的子任务

【完成条件】

- 文件成功写入 task/
- 输出创建成功的文件路径
- 不做任何进一步动作
